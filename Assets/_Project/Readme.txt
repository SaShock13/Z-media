Архитектура
Проект реализован в подходе MVC .
Цель архитектуры — сделать код расширяемым и поддерживаемым: чтобы добавление новых форм, цветов, механик поведения 
или UI не приводило к переписыванию половины проекта.

Model слой (данные и правила)
Model содержит только данные и логику, которая не зависит от Unity.
Основные классы Model:
UnitModel — описание юнита: id, команда, выбранные Shape/Color/Size, итоговые характеристики (Stats)
Stats — структура характеристик (HP, ATK, SPEED, ATKSPD)
ArmyModel — армия: список юнитов, TeamId, выбранная формация
BattleModel — модель боя, содержит две армии и проверяет завершение боя
Model слой не содержит MonoBehaviour, не обращается к Transform, не создаёт префабы и не знает про UI.

View слой (визуализация):
View отвечает за внешний вид и отображение.
Основные классы View:
UnitView — компонент на префабе юнита (Cube или Sphere), отвечает за: цвет (через материал), масштаб (визуальный размер),ориентацию/флип по направлению.
View ничего не решает сам: он не выбирает цели, не считает урон и не хранит логику боя.

Controller слой (управление):
Controller соединяет Model и View и управляет поведением.
Основные контроллеры:
UnitController — логика одного юнита: поиск цели (через TargetSelector), движение к цели, атака с учётом ATKSPD , 
нанесение урона в модель цели (target.Model.Stats.TakeDamage(...)).
ArmyController — управляет армией: перестроение в выбранную формацию, применение формации к существующим UnitView.
BattleController (или компонент сцены) — создаёт бой, запускает симуляцию, отслеживает конец боя и переключает UI/состояния.

Создание юнитов и армий (UnitFactory / ArmyFactory)
Чтобы логика генерации не была размазана по сцене и контроллерам, создание игровых сущностей вынесено в фабрики:
UnitFactory — создаёт UnitModel. Получает UnitDatabase (набор доступных форм, цветов, размеров и базовых статов) и seed (для рандома).
При создании юнита фабрика выбирает параметры юнита (форма, цвет, размер), рассчитывает итоговые характеристики: 
берёт базовые статы и применяет модификаторы Shape/Color/Size, создаёт UnitModel с уникальным Id.
Вся логика генерации юнита находится в одном месте, легко расширять (например, сделать пресеты, редкие формы, баланс), использование seed позволяет 
воспроизводить одинаковый бой (предопределенная генерация).

ArmyFactory  — создаёт ArmyModel и наполняет её юнитами через UnitFactory. Использует UnitFactory и отвечает за создание армии как набора данных.
При создании армии создаётся ArmyModel (teamId, список юнитов, выбранная формация), в цикле добавляется нужное количество UnitModel.
Создание армии отделено от Unity и не зависит от сцены, спавн не дублируется в разных местах, проще добавлять новые режимы .
Фабрики создают только Model-объекты (UnitModel, ArmyModel).Unity-префабы (UnitView) создаются отдельно в Unity-слое (в SpawnArmy), где происходит связывание Model ↔ View.

Генерация характеристик (Shape / Size / Color)
Все характеристики считаются одинаковым способом:
Берётся базовый Stats (HP=100, ATK=10, SPEED=10, ATKSPD=1), применяются модификаторы: формы (UnitShapeSO), размера (UnitSizeSO), цвета (UnitColorSO).
Каждый модификатор хранится в ScriptableObject, что позволяет добавлять новые параметры без переписывания кода.

Дополнительная фича: Формации
В качестве доп фичи реализована система формаций армии.
Армия может быть построена разными способами (например): Line, Box, Wedge, Chaotic.
Формация хранится в ArmyModel, а применение формации выполняет ArmyController.
На UI расположены кнопки для выбора формации каждой армии (отдельный блок кнопок для армии A и армии B).
При нажатии кнопки меняется ArmyModel.Formation, вызывается ArmyController.ApplyFormation(...), юниты перестраиваются в реальном времени.

Логика боя:
Юнит выбирает цель из армии противника (ближайший живой), юнит движется к цели со скоростью SPEED,Когда дистанция становится меньше attackRange, юнит начинает бить.
При HP ≤ 0 юнит считается мёртвым и удаляется с поля.

Завершение боя
BattleModel проверяет условие окончания:если в одной армии не осталось живых юнитов - бой завершён.
Вызывается событие BattleFinished, которое обрабатывает контроллер и возвращает игрока в главное меню.

Добавление новых форм: 
Форма — это UnitShapeSO, который содержит : модификаторы характеристик, ссылку на prefab.
Чтобы добавить новую форму нужно создать новый prefab, создать новый UnitShapeSO, добавить его в UnitDatabase.

Добавление нового цвета: 
Цвет — это UnitColorSO, который содержит: модификаторы характеристик и ViewColor (цвет отображения).
Чтобы добавить новый цвет нужно создать новый UnitColorSO, заполнить параметры и добавить его в UnitDatabase.

Время выполнения: Время, потраченное на выполнение тестового задания - 12 часов.
